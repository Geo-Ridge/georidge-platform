## Context

Viewer app has 17 migration files accumulated over development. Django's `squashmigrations` has already generated a single replacement file. The originals need to be deleted and the squashed file renamed.

## Goals / Non-Goals

**Goals:**
- Delete 17 original viewer migration files
- Rename `0001_initial_squashed_0017_remove_maponly_fix_dark_banner.py` to `0001_initial.py`

**Non-Goals:**
- Squashing other apps (accounts, audit, projects, datasources)
- Modifying the squashed file's content

## Decisions

### 1. Use Django-generated squashed file

The file was generated by `manage.py squashmigrations viewer 0001 0017`. It contains:
- `replaces = [...]` listing all 17 original migration names
- 3 `CreateModel` operations (BaseMap, ThemeProfile, LayerSearchConfig)
- All `RunPython` data operations eliminated

### 2. Delete originals, don't keep as empty shells

With the squashed file's `replaces` list, Django can handle existing databases. The empty shells serve no purpose.

## Risks / Trade-offs

- **Existing databases** → The `replaces` list ensures Django marks all 17 as applied when it sees the squashed file. No data loss.
- **New databases** → Django runs only the squashed file. Clean start with 3 operations.
- **File deletion is irreversible in git** → But the squashed file contains everything. Git history preserves the originals.
